Microservices API Portal

4 independent microservices built with Clean Architecture, CQRS, JWT RS256 authentication, RabbitMQ messaging, and gRPC. Each service owns its own PostgreSQL database.

.NET 8 Clean Architecture CQRS + MediatR JWT RS256 BCrypt 12 RabbitMQ + MassTransit gRPC + Protobuf PostgreSQL × 4 EF Core 8 YARP Gateway xUnit + Moq · 59 tests
How to use this portal
1
Register or log in via Auth Service
Open Auth Service in the sidebar. Use POST /api/auth/register to create an account or POST /api/auth/login if you already have one.
{ "email": "you@example.com", "password": "Password1!", "firstName": "John", "lastName": "Doe" }
2
Copy your Access Token
The response returns an accessToken — a signed RS256 JWT valid for 15 minutes — and a refreshToken for rotation.
3
Paste the token and click ⚡ Apply to all
Paste the accessToken into the JWT panel in the sidebar. Click ⚡ Apply to all — it's injected as Authorization: Bearer … on every request across all services.
4
Explore the APIs
Products catalog is public — no token needed. Orders and Payments require a valid JWT. Admin-only endpoints (create/delete products) require the Admin role.
Services
🔐
Auth Service
:5004 · RS256 · BCrypt 12 · Rate Limited
Identity management with refresh token rotation, asymmetric RSA signing, BCrypt factor 12, and IP-based rate limiting (5 req/min).
POST /registerPOST /login POST /refreshPOST /revokeGET /profile
📋
Orders Service
:5001 · CQRS · gRPC · MassTransit
Full order lifecycle. Checks stock via gRPC, publishes OrderCreated to RabbitMQ, listens for PaymentProcessed to confirm or cancel.
POST /ordersGET /orders/{id} GET /orders/customerDELETE /orders/{id}
📦
Products Service
:5002 · gRPC server · Seeded catalog
Product catalog with 10 seeded items across 3 categories. Public read, Admin-only writes. Exposes gRPC for stock verification by Orders.
GET /productsGET /products/{id} POST /productsPATCH /stockDELETE /{id}
💳
Payments Service
:5003 · RabbitMQ consumer
Processes payments triggered by OrderCreated events. Publishes PaymentProcessed back to RabbitMQ — Orders listens and confirms or cancels.
POST /paymentsGET /payments/{orderId}
