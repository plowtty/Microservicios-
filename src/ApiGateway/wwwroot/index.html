Microservices API Portal

Explore and test 4 independent microservices built with Clean Architecture, CQRS, JWT RS256 authentication, RabbitMQ messaging, and gRPC communication.

.NET 8 Clean Architecture CQRS + MediatR JWT RS256 BCrypt 12 RabbitMQ gRPC PostgreSQL EF Core 8 YARP Gateway
How to use this portal
1
Register or log in via Auth Service
Open Auth Service → POST /api/auth/register to create a new account, or POST /api/auth/login if you already have one.
{ "email": "you@example.com", "password": "Password1!", "firstName": "John", "lastName": "Doe" }
2
Copy your Access Token
The response will contain an accessToken. Copy it from the response body — it's a signed RS256 JWT valid for 15 minutes.
3
Paste the token in the sidebar and click Apply
Paste the token in the JWT panel on the left. Click ⚡ Apply to all — it will be injected as a Bearer header automatically on every request across all services.
4
Explore the APIs below
Browse the Products catalog freely (public). Place orders, track payments — all protected endpoints now authenticate with your token automatically.
Services
🔐
Auth Service
:5004 · JWT RS256 · BCrypt 12
Identity & access management with refresh token rotation, RS256 asymmetric signing, and rate limiting.
POST /register POST /login POST /refresh POST /revoke GET /profile
📋
Orders Service
:5001 · CQRS · MediatR · MassTransit
Order lifecycle management. Validates stock via gRPC, triggers async payment flow via RabbitMQ events.
POST /orders GET /orders/{id} GET /orders/customer DELETE /orders/{id}
📦
Products Service
:5002 · gRPC · Seeded catalog
Product catalog with 10 seeded items. Public read access, Admin-only writes. Exposes a gRPC endpoint for stock verification.
GET /products GET /products/{id} POST /products PATCH /stock
💳
Payments Service
:5003 · RabbitMQ consumer
Processes payments triggered by OrderCreated events. Publishes PaymentProcessed events consumed by Orders to confirm or cancel.
POST /payments GET /payments/{orderId}
